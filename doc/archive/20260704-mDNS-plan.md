# Bonjour/DNS-SD autodiscovery for `tp server`

## Context

When `tp server` is running, the LX-D02 should appear automatically in macOS **Printers & Scanners → Add Printer** as a Bonjour device, letting the user add it (choosing the PPD from `ippsrv/ppd/LX-D02.ppd` if needed). macOS discovers network printers by browsing DNS-SD `_ipp._tcp` over mDNS, so the server must advertise itself.

A prior attempt exists on the `mdns` branch and failed for identifiable reasons; this plan replaces it (implement fresh on a new branch off `master`, don't merge `mdns`):

- **Wrong port**: advertised hardcoded `6031` while the server default is `6310` (transposed digits), plus hardcoded `localhost` in `adminurl`.
- **Wrong resource path**: `rp=ipp/print`, but the real IPP endpoint is `POST /printers/{name}` → must be `rp=printers/default`.
- **Library**: `grandcat/zeroconf` is unmaintained and notoriously fails same-host discovery on macOS (doesn't answer mDNSResponder's unicast queries) → use `github.com/brutella/dnssd` (RFC 6762/6763-compliant).
- **Half-claimed AirPrint**: `pdl=...,image/urf`, lowercase `urf=`, bogus `AirPrint=none` — the server can only render `application/pdf`. Advertise plain IPP/PDF only.
- **Wrong lifecycle**: registered inside `newBasicIPPServer` before any listener existed.

User decisions (confirmed):
- Change default `-addr` to `:6310` (all interfaces) so discovery works out of the box.
- mDNS **on by default**, with a `-no-mdns` opt-out flag.

## Changes

### 1. Dependency
Add `github.com/brutella/dnssd` to `go.mod` (`go get github.com/brutella/dnssd`).

### 2. New file: `ippsrv/bonjour.go`
- `WithBonjour() Option` — matches existing option idiom (`WithDebug`, `WithDumpDir` in `ippsrv/http.go:45-63`); sets `s.bonjour.enabled = true`.
- Pure, testable TXT builder:
  ```go
  // hostname is the mDNS host label without ".local"
  func txtRecord(p PrinterInformer, baseURL, hostname string, port int) map[string]string
  ```
  Keys: `txtvers=1`, `qtotal=1`, `rp` = `strings.TrimPrefix(path.Join(baseURL, p.Name()), "/")` → `printers/default` (derived, never hardcoded), `ty=p.MakeAndModel()`, `note=p.Info()`, `pdl=application/pdf` (only — no URF), `adminurl=http://<host>.local.:<port>/admin/`, `UUID=p.UUID()` (bare, no `urn:uuid:` prefix in TXT), `Color=F`, `Duplex=F`, `priority=0`.
- `localHostname()` — `os.Hostname()` with `.local`/`.local.` suffix stripped (macOS returns `Name.local`).
- `(s *Server) startBonjour(ta *net.TCPAddr) error`:
  - If `ta.IP.IsLoopback()`: return error ("printer would be advertised as <host>.local:PORT but unreachable there; use -addr :PORT") — caller logs it as a warning and continues serving without mDNS.
  - One `dnssd.Service` per printer in `s.pp` via `dnssd.NewService(dnssd.Config{Name, Type: "_ipp._tcp", Domain: "local", Port, Text})`; instance name = `p.MakeAndModel()`, or `"MakeAndModel (Name)"` when `len(s.pp) > 1`. If bound IP is specific (not unspecified), set `cfg.IPs = []net.IP{ta.IP}`.
  - `dnssd.NewResponder()` → `responder.Add(service)` → goroutine running `responder.Respond(ctx)` (blocks; handles probing, conflict renaming, and goodbye packets on cancel). Store `cancel` + `done` channel on `s.bonjour`.
  - No `_ipps` (no TLS) and no `_universal._sub._ipp._tcp` / `URF=` (no URF rendering) — plain IPP printer; user picks driver/PPD in the "Use" dropdown.

### 3. `ippsrv/http.go`
- `Server` struct: add `bonjour struct { enabled bool; cancel context.CancelFunc; done chan struct{} }`.
- Refactor `ListenAndServe` (line 223) to `net.Listen("tcp", addr)` + `s.srv.Serve(l)` so the actual bound `*net.TCPAddr` is known; keep setting `s.srv.Addr = addr` (used by `Info()`, line 115). When `bonjour.enabled`, call `startBonjour(l.Addr().(*net.TCPAddr))`; on error, `slog.Warn` and continue. (`Serve` returns `http.ErrServerClosed` on shutdown, same as before — cmdserver's error filter at `cmdserver.go:78` is unaffected.)
- `Shutdown` (line 228): before the existing shutdown loop, if `bonjour.cancel != nil`, cancel and wait on `done` (or `sctx.Done()`) so goodbye packets go out while the process is alive.

### 4. `cmd/tp/internal/cmdserver/cmdserver.go`
- Change `-addr` default `localhost:6310` → `:6310`; fix its help text (currently the placeholder "custom flag is different than the global flags") and the placeholder `Long` description — mention that binding a loopback address disables discovery.
- Add `CmdServer.Flag.BoolVar(&noMDNS, "no-mdns", false, "disable Bonjour/DNS-SD printer advertisement")`.
- In `runServer`: `if !noMDNS { opts = append(opts, ippsrv.WithBonjour()) }`. No addr parsing here — loopback detection happens in `startBonjour` off the real bound address (covers `localhost`, `127.0.0.1`, `[::1]` uniformly).

### 5. Minimal IPP attribute fixes in `ippsrv/ipp.go` (macOS probes the printer immediately after discovery)
1. `printer-uri-supported` (line ~144) currently emits the bare path `/printers/`. Fix: extract `printer-uri` from the request operation attributes in `handleGetPrinterAttributes` (same `extractValue[goipp.String]` pattern as `printerFromRequest`, ipp.go:194) and echo it; fall back to `ih.baseURL + p.Name()`.
2. `ipp-versions-supported`: `"1.1", "2.0"` (server already responds with goipp DefaultVersion 2.0 and dispatches by opcode).
3. `printer-uuid` (line ~175): prefix with `urn:uuid:` (it's `TagURI`, RFC 8011 §5.4.34); TXT `UUID=` stays bare per Bonjour Printing spec.

Deferred (out of scope): URF/PWG-raster support, `media-*` keyword normalization, honoring `requested-attributes`, embedding the PPD.

### 6. Tests — new `ippsrv/bonjour_test.go` (testify, matching existing test style)
- Table test for `txtRecord`: `rp=printers/default` derived from `("/printers/", "default")` (regression for the branch bug); required keys present; `pdl` is exactly `application/pdf`; no `urf`/AirPrint keys; `adminurl` format.
- `localHostname()` suffix stripping.
- Loopback rejection for `127.0.0.1`/`[::1]`; acceptance of unspecified/LAN addresses (extract a `checkAdvertisable(*net.TCPAddr) error` helper if that's cleaner to test).
- No network I/O in unit tests.

## Verification

1. `go test ./...` and `go build ./...` from repo root.
2. `go run ./cmd/tp server -addr localhost:6310` → warning logged, server still serves, no advertisement.
3. `go run ./cmd/tp server` (now binds `:6310`; accept the macOS firewall prompt if shown).
4. `dns-sd -B _ipp._tcp local.` → "Thermal Printer" appears; `dns-sd -L "Thermal Printer" _ipp._tcp local.` → SRV `<host>.local.:6310`, TXT `rp=printers/default`, `pdl=application/pdf`, `UUID=...`.
5. System Settings → Printers & Scanners → Add Printer → printer appears in the Default/Bonjour list. Add it (select the PPD `ippsrv/ppd/LX-D02.ppd` via "Use → Other…" if macOS's generic choice misbehaves); run with `-v -dumpdir <dir>` to capture what macOS sends.
6. Print a test page (PDF path).
7. Ctrl-C the server → `dns-sd -B _ipp._tcp local.` shows the instance removed promptly (goodbye packets).

## Risks

- Modern macOS prefers AirPrint/IPP-Everywhere; a bare `_ipp._tcp` service will be listed, but driver auto-selection may degrade to "Generic" — the PPD selection covers this. If setup stalls, the protocol dumps show which attributes macOS wants; real URF/PWG-raster is a possible follow-up, deliberately not claimed now.
- Binding `0.0.0.0` by default exposes the unauthenticated IPP server to the LAN (user accepted this trade-off; `-addr localhost:6310` remains available).
- macOS firewall may block UDP 5353 answers or IPP connections for unsigned dev binaries — watch for the allow prompt.
